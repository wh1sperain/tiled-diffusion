## 1. 项目概述与研究动机

### 1.1 研究问题

图像拼贴（Image Tiling）——将多张图像无缝拼接形成连续视觉场景——在纹理创建、游戏资产开发、数字艺术等领域至关重要。传统方法依赖人工制作，存在严重的可扩展性和灵活性限制。

### 1.2 现有方法的局限

现有基于生成模型的方法主要集中在**单一纹理图像的平铺**，缺乏对**多图像互联拼接**（many-to-many tiling）的支持。

### 1.3 本文贡献

Tiled Diffusion 提出了一种在扩散模型（Diffusion Models）的**潜在空间（Latent Space）**中施加拼接约束的方法，支持：
- **Self-tiling**：单张图像与自身在 X/Y 轴上无缝拼接
- **One-to-one**：两张图像之间的无缝拼接
- **Many-to-many**：多张图像之间任意方向的无缝拼接

核心创新：在去噪过程的每一步中，通过 **Padding 区域交换** 和 **相似性约束** 保证相邻 tile 边界一致。

## 2. 核心思想与算法原理

### 2.1 潜在空间中的 Padding 机制

核心思想：在潜在空间 tensor 的四周扩展 padding 区域（宽度为 `max_width`），用于存储与相邻 tile 的**上下文信息**。

```
┌──────────────────────────────┐
│          padding (Up)        │
│  ┌────────────────────────┐  │
│  │                        │  │
│p │    原始内容区域          │p │
│a │    (height×width)       │a │
│d │                        │d │
│  └────────────────────────┘  │
│         padding (Down)       │
└──────────────────────────────┘
 Left                     Right
```

- 每个 tile 的 padding 区域在去噪的每一步都会被**相邻 tile 的边缘内容覆盖**
- 这样 U-Net 在做去噪预测时，能"看到"相邻 tile 的上下文，从而生成视觉一致的边界

### 2.2 三步约束机制

每一个去噪时间步中执行以下三步：

1. **Similarity Constraint（相似性约束）**：共享同一拼接边的多个 tile 之间，将有一个 tile 的边缘内容（`max_replica_width` 宽度）复制给其他 tile 对应位置，确保它们在拼接边附近生成相似内容。
2. **Tiling（Padding 交换）**：将每个 tile 的 padding 区域替换为其相邻 tile 的对应边缘内容，使 U-Net 可以获取跨 tile 的上下文信息。
3. **Random Padding Constraint（随机 Padding 约束）**：推理结束后，额外执行一次，随机选取一个源 tile 的 padding 内容同步到共享同边的其他 tile，消除最终结果中 padding 区域残留的不一致。

### 2.3 旋转矩阵

当两个 tile 以不同方向的边相邻时（如一个用右边拼接另一个的上边），需要对复制的 tensor 切片做旋转（`torch.rot90`）。项目定义了两个旋转矩阵：
- `SIMILARITY_ROTATION_MATRIX`：用于 Similarity Constraint 中校正方向
- `TILING_ROTATION_MATRIX`：用于 Tiling 交换中校正方向

### 2.4 图约束（Graph-based Groups）

通过 `side_id` 和 `side_dir`（`cw`/`ccw`）构建约束图：
- `side_id` 相同且 `side_dir` 相反的两个边为一对约束边
- `generate_graph_groups()` 将所有约束关系组织为 `groups` 字典，key 为 `"{source_latent_idx}_{source_side_idx}"`

## 4. 数据处理

### 4.1 图像预处理 (`utils.py: preprocess()`)

```python
def preprocess(image):
    if isinstance(image, torch.Tensor):
        return image                    # 若已是 Tensor 则直接返回
    elif isinstance(image, PIL.Image.Image):
        image = [image]                 # 统一为 list 处理

    if isinstance(image[0], PIL.Image.Image):
        w, h = image[0].size
        w, h = (x - x % 8 for x in (w, h))  # 确保宽高是 8 的倍数（VAE 下采样要求）

        image = [np.array(i.resize((w, h), resample=PIL_INTERPOLATION["lanczos"]))[None, :] for i in image]
        # 使用 Lanczos 重采样缩放 → numpy 数组 → 增加 batch 维度

        image = np.concatenate(image, axis=0)      # 沿 batch 拼接
        image = np.array(image).astype(np.float32) / 255.0  # 归一化到 [0, 1]
        image = image.transpose(0, 3, 1, 2)        # NHWC → NCHW
        image = 2.0 * image - 1.0                  # [0,1] → [-1, 1]（diffusion 模型标准输入范围）
        image = torch.from_numpy(image)             # numpy → Tensor
```

**为什么宽高必须是 8 的倍数？**  
因为 VAE 编码器有 3 层下采样（每层 stride=2），总下采样倍率为 $2^3 = 8$。如果输入尺寸不是 8 的倍数，下采样后的大小将无法正确还原。

**为什么归一化到 [-1, 1]？**  
Stable Diffusion 的 VAE 训练时使用 [-1, 1] 作为输入值域。

### 4.2 文本编码 (`latent_class.py: set_text_embs()`)

```python
def set_text_embs(self, tokenizer, text_encoder):
    # 正向提示词编码
    tokenized_prompts = tokenizer(
        [self.prompt],
        max_length=tokenizer.model_max_length,  # CLIP 最大长度 77 tokens
        padding='max_length',                    # 不足长度补 padding
        truncation=True,                         # 超长截断
        return_tensors='pt',                     # 返回 PyTorch Tensor
    )
    with torch.no_grad():
        cond_embeddings = text_encoder(tokenized_prompts.input_ids.to('cuda'))[0]
        # 形状: [1, 77, 768]（CLIP hidden_size=768）

    # 负向提示词编码（同样流程）
    tokenized_prompts = tokenizer(
        [self.negative_prompt],
        max_length=tokenizer.model_max_length,
        padding='max_length',
        truncation=True,
        return_tensors='pt',
    )
    with torch.no_grad():
        uncond_embeddings = text_encoder(tokenized_prompts.input_ids.to('cuda'))[0]

    # 拼接 [uncond, cond]  → 形状 [2, 77, 768]
    self.text_embeddings = torch.cat([uncond_embeddings, cond_embeddings])
```

**为什么将 uncond 和 cond 拼接？**  
Classifier-Free Guidance (CFG) 要求同时对无条件和有条件输入做前向传播，然后计算加权差值。拼接后可以一次 batch forward 得到两个预测结果。

### 4.3 Latent 初始化

**纯文本生成（txt2img）**：生成随机高斯噪声 tensor 作为初始 latent。

```python
def set_latents(self, generator, in_channels=4, max_width=10):
    if self.is_xy():  # 四边拼接：需要在 H 和 W 两个维度都加 padding
        self.pre_latent = torch.randn(
            (1, in_channels, self.height // 8 + 2 * max_width, self.width // 8 + 2 * max_width),
            generator=generator, device='cuda',
        )
```

- `height // 8` 和 `width // 8`：潜在空间尺寸 = 图像空间尺寸 / 8
- `+ 2 * max_width`：在两侧各加 `max_width` 个 padding

**图像引导生成（img2img）**：通过 VAE 编码源图像并添加噪声。

```python

# scaling_factor 通常为 0.18215，对齐训练时的 latent 分布
```

## 5. 模型部署与加载

### 5.1 模型组件

项目基于 **Stable Diffusion v1.5** (`runwayml/stable-diffusion-v1-5`)，包含以下组件：

| 组件 | 类型 | 作用 | 参数量级 |
|------|------|------|---------|
| **Tokenizer** | `CLIPTokenizer` | 文本分词器，将文本转为 token ids | - |
| **Text Encoder** | `CLIPTextModel` | 将 token ids 编码为 768 维文本特征 | ~123M |
| **U-Net** | `UNet2DConditionModel` | 去噪网络核心，预测噪声 | ~860M |
| **VAE** | `AutoencoderKL` | 变分自编码器，像素空间 ↔ 潜在空间 | ~84M |
| **Scheduler** | DDPM / DDIM / Euler | 控制去噪时间步和噪声调度 | - |

### 5.2 模型加载代码

```python
class SDLatentTiling:
    def __init__(self, model_id=MODEL_ID_SD_1_5, scheduler="ddpm", local_files_only=True):
        pretrained_kwargs = {"local_files_only": local_files_only}
        # 设置环境变量，避免联网
        if local_files_only:
            os.environ.setdefault("HF_HUB_OFFLINE", "1")
            os.environ.setdefault("TRANSFORMERS_OFFLINE", "1")

        self.tokenizer = CLIPTokenizer.from_pretrained(model_id, subfolder='tokenizer', **pretrained_kwargs)
        self.text_encoder = CLIPTextModel.from_pretrained(model_id, subfolder='text_encoder', **pretrained_kwargs).to('cuda')
        self.unet = UNet2DConditionModel.from_pretrained(model_id, subfolder='unet', **pretrained_kwargs).to('cuda')
        self.vae = AutoencoderKL.from_pretrained(model_id, subfolder='vae', **pretrained_kwargs).to('cuda')
```

**为什么使用 `local_files_only=True`？**  
在离线环境（如实验室无网络环境）中，从本地 Hugging Face 缓存加载模型权重，避免运行时联网下载。

**关于 `.to('cuda')`**  
将模型参数（权重）从 CPU 内存移动到 GPU 显存，以便利用 GPU 加速推理。

### 5.3 调度器选择

```python
if scheduler == "ddpm":
    self.scheduler = DDPMScheduler.from_pretrained(...)     # Denoising Diffusion Probabilistic Models
elif scheduler == "ddim":
    self.scheduler = DDIMScheduler.from_pretrained(...)     # Denoising Diffusion Implicit Models (更快)
else:
    self.scheduler = EulerDiscreteScheduler.from_pretrained(...)  # Euler 方法
```

- **DDPM**：原始的马尔可夫去噪过程，推理步数多（通常 1000 步），质量好但慢。项目中使用 40 步的缩减版本。
- **DDIM**：非马尔可夫过程，允许跳步，可以用更少的步数获得类似质量。
- **Euler**：是 ODE 求解器方法的离散化版本，速度和质量平衡。

## 6. 关键参数设置与含义

| 参数 | 默认值 | 含义 | 影响 |
|------|--------|------|------|
| `inference_steps` | 40 | 去噪迭代步数 | 步数越多生成质量越好，但耗时线性增长 |
| `cfg_scale` | 7.5 | Classifier-Free Guidance 系数 | 越大越遵循 prompt 但可能过饱和；通常 7~12 |
| `height` / `width` | 512 | 生成图像的像素尺寸 | SD 1.5 训练分辨率为 512×512 |
| `max_width` | 16 (实验) / 32 (demo) | Padding 区域宽度（潜在空间像素） | 越大上下文越丰富但显存消耗成平方增长 |
| `max_replica_width` | 3~5 | 相似性约束的复制宽度 | 越大拼接越连续但可能降低图像多样性 |
| `strength` | 0.95 | img2img 强度（0~1） | 越高加噪越多、生成结果偏离源图越大 |
| `seed` | 各例不同 | 随机种子 | 保证可复现性 |
| `blend_mode` | "overwrite" | Padding 覆盖方式 | "overwrite" 或 "weighted"（线性融合） |
| `scheduler` | "ddpm" | 噪声调度器类型 | 影响去噪质量和速度 |
| `negative_prompt` | "blured, ugly..." | 负向提示词 | 引导模型避免低质量输出 |

### 6.1 `max_width` 的物理含义

在潜在空间中，1 个像素对应图像空间 8 个像素。因此：
- `max_width = 16` → padding 覆盖图像空间 $16 \times 8 = 128$ 像素
- `max_width = 32` → padding 覆盖图像空间 $32 \times 8 = 256$ 像素（即图像宽度的一半）

**trade-off**：padding 越大，U-Net 看到的跨 tile 上下文越多，拼接越自然；但计算量和显存消耗增加。

### 6.2 `max_replica_width` 的作用

相似性约束只复制 `max_replica_width` 宽度的边缘带，这是 padding 内侧的一个子区间：
- 太小（如 0）：多个共享同边的 tile 无一致性约束，拼接处可能有明显差异
- 太大：约束太强，限制了图像的多样性

### 6.3 `side_id` 和 `side_dir` 的约束建模

`LatentClass` 的 `side_id` 和 `side_dir` 列表长度固定为 4，分别对应 **[Right, Left, Up, Down]**：

```python
# Self-tiling（四边都与自身拼接）
side_id  = [1, 1, 2, 2]
side_dir = ['cw', 'ccw', 'cw', 'ccw']
# 含义：Right 标记为 id=1 方向 cw，Left 标记为 id=1 方向 ccw   → 右左互联
#        Up 标记为 id=2 方向 cw，Down 标记为 id=2 方向 ccw      → 上下互联
```

- `side_id` 相同 + `side_dir` 相反 → 两条边形成拼接约束
- `side_id = None` → 该方向不参与拼接

### 7.2 LatentClass 构建

```python
class LatentClass:
    def __init__(self, prompt, negative_prompt="", height=512, width=512,
                 side_id=[], side_dir=[], is_fixed=False, source_image=None, eval_image=None):
        self.prompt = prompt                    # 正向提示词
        self.negative_prompt = negative_prompt  # 负向提示词
        self.height = height                    # 目标图像高度
        self.width = width                      # 目标图像宽度
        if len(side_id) == 0:                   # 四方向拼接配置
            self.side_id = [None, None, None, None]   # 默认不拼接
            self.side_dir = [None, None, None, None]
        else:
            self.side_id = side_id
            self.side_dir = side_dir
        self.is_fixed = is_fixed                # 是否在去噪过程中锁定
        self.text_embeddings = None             # 文本 embedding（延迟计算）
        self.pre_latent = None                  # 约束前的 latent（当前步输入）
        self.post_latent = None                 # 约束后的 latent（用于 U-Net 输入）
        self.image = None                       # 最终解码图像
        self.source_image = source_image        # img2img 的源图像
        self.eval_image = eval_image            # 评估参考图像
```

**`pre_latent` 与 `post_latent` 的区别**：
- `pre_latent`：存储每步去噪后的"原始"结果，以及 similarity constraint 作用后的结果
- `post_latent`：在 `pre_latent` 基础上，经 tiling（padding 交换）后形成的 tensor，作为下一步 U-Net 的输入
- 两个分开存储是为了避免约束操作污染原始数据

### 7.4 核心推理流程 `SDLatentTiling.__call__`

这是整个项目最核心的函数，下面逐步详解：

#### Step 1：文本编码

```python
for latent in latents_arr:
    latent.set_text_embs(tokenizer=self.tokenizer, text_encoder=self.text_encoder)
```
**作用**：为每个 tile 计算文本 embedding（正向 + 负向），形状 `[2, 77, 768]`。  
**原因**：不同 tile 可以有不同的 prompt，因此每个 tile 独立编码。

#### Step 2：初始化随机生成器

```python
generator = torch.Generator(device='cuda')
generator.manual_seed(seed)
```
**作用**：创建 CUDA 随机数生成器并设定种子。  
**原因**：保证实验可复现。所有随机操作（噪声生成等）共享同一 generator。

#### Step 3：Latent 初始化

```python
for latent in latents_arr:
    if latent.source_image is not None:
        # ==== img2img 路径 ====
        image = preprocess(latent.source_image)
        # 预处理图像：resize → 归一化 → [-1,1] → Tensor

        timesteps, num_inference_steps = retrieve_timesteps(self.scheduler, inference_steps, device, None)
        timesteps, num_inference_steps = get_timesteps(self.scheduler, num_inference_steps, strength, device)
        # 根据 strength 计算真正的起始时间步
        # strength=0.95 意味着从 95% 噪声开始，只保留 5% 原图信息

        latent_timestep = timesteps[:1].repeat(1)
        latents = self.prepare_latents(image, latent_timestep, 1, 1,
```

```python
def generate_graph_groups(latents_arr):
    groups = {}
    for latent_idx, latent in enumerate(latents_arr):
        side_id = latent.side_id       # 如 [1, 1, 2, 2]
        side_dir = latent.side_dir     # 如 ['cw', 'ccw', 'cw', 'ccw']
        for side_idx, side_val in enumerate(side_id):
            if side_val is None:
                continue
            # 找到方向相反的匹配边
            current_side_dir = side_dir[side_idx]
            opposite_side_dir = 'ccw' if current_side_dir == 'cw' else 'cw'

            # 遍历所有 tile 找匹配
            for latent2_idx, latent2 in enumerate(latents_arr):
                for target_side_idx, (target_side, target_dir) in enumerate(zip(latent2.side_id, latent2.side_dir)):
                    if target_side == side_val and target_dir == opposite_side_dir:
                        key = f"{latent_idx}_{side_idx}"
                        if key not in groups:
                            groups[key] = {'target_latent_idx': [], 'target_side_idx': []}
                        groups[key]['target_latent_idx'].append(latent2_idx)
                        groups[key]['target_side_idx'].append(target_side_idx)
    return groups
```

- key `"0_0"` (tile 0 的右边) → targets: tile 0 的左边
- key `"0_1"` (tile 0 的左边) → targets: tile 0 的右边
- 依此类推

```python
    torch.cuda.empty_cache()  # 释放未使用的 GPU 缓存
    gc.collect()               # 触发 Python 垃圾回收

    latent_model_input = torch.cat([latent.post_latent] * 2)
    # [post_latent, post_latent] → batch=2，分别给 uncond 和 cond 用
    # 与 text_embeddings 的 [uncond_emb, cond_emb] 一一对应

    latent_model_input = self.scheduler.scale_model_input(latent_model_input, timestep=t)
    # 某些 scheduler（如 Euler）需要根据时间步缩放输入

    with torch.no_grad():
        noise_pred = self.unet(
            latent_model_input, t,
            encoder_hidden_states=latent.text_embeddings
        ).sample
    # U-Net 预测噪声，输入：
    #   - latent_model_input: (2, 4, H', W')，含 padding
    #   - t: 当前时间步标量
    #   - encoder_hidden_states: (2, 77, 768) 文本 embedding
    # 输出：noise_pred (2, 4, H', W')

    noise_pred_uncond, noise_pred_cond = noise_pred.chunk(2)
    noise_pred = noise_pred_uncond + cfg_scale * (noise_pred_cond - noise_pred_uncond)
    # Classifier-Free Guidance:
    # ε_guided = ε_uncond + w * (ε_cond - ε_uncond)
    # w = cfg_scale，越大越遵循 prompt

    latent_new = self.scheduler.step(noise_pred, t, latent.post_latent).prev_sample
    # 去噪一步
    # DDPM: x_{t-1} = (1/√α_t) * (x_t - (β_t/√(1-ᾱ_t)) * ε_pred) + σ_t * z
    # 其中 z 是随机噪声（DDPM 有随机性），DDIM 则是确定性的

    latent.pre_latent = latent_new
    latent.clone_post_latents()
    # 更新 pre_latent 为去噪结果，并克隆到 post_latent
    # 为下一步的 similarity constraint 和 tiling 做准备
```

### 7.5 LatentHandler 的三大核心操作

#### 7.5.1 `apply_similarity_constraint()` — 相似性约束

```python
@staticmethod
def apply_similarity_constraint(latents_arr, step, groups, max_width=10, max_replica_width=5):
    for key, val in groups.items():
        target_latent_idx = val['target_latent_idx']
        target_side_idx = val['target_side_idx']
        len_target = len(target_latent_idx)
        if len_target <= 1:
            continue     # 只有一个 target 无需约束

        # 轮询选择一个源 tile（通过 step 取模）
        candidate_idx = step % len_target
        source_latent_idx = target_latent_idx[candidate_idx]
        source_side_idx = target_side_idx[candidate_idx]

        latent = latents_arr[source_latent_idx]
        tensor_slice = get_tensor_slice_from_latent_and_side_idx(
            latent=latent.pre_latent, side_idx=source_side_idx,
            max_width=max_width, max_replica_width=max_replica_width
        )
        # 从源 tile 的 pre_latent 中切出 padding 内侧的 max_replica_width 宽条带
        # 这个区域是"真实内容区域的边缘"

        for lat_idx, side_idx in zip(target_latent_idx, target_side_idx):
            if source_latent_idx == lat_idx and source_side_idx == side_idx:
                continue    # 跳过自身

            rotation_value = SIMILARITY_ROTATION_MATRIX[source_side_idx][side_idx]
            rotated_tensor_slice = torch.rot90(tensor_slice, rotation_value, [2, 3])
            # 根据源和目标的方向差异旋转切片

            current_latent = latents_arr[lat_idx]
            # 将旋转后的切片写入目标 tile 的对应位置
            if side_idx == 0:  # Right
                current_latent.pre_latent[:, :, :,
                    W - max_width - max_replica_width: W - max_width] = rotated_tensor_slice
            # ... 其他方向类推
```

**原理**：在 many-to-many 场景中，多个 tile 共享同一拼接约束（如 tile A 的右边和 tile B 的右边都标记为 `side_id=1, cw`），此约束确保这些 tile 在拼接边附近生成相似的内容。

**轮询机制**：每步选不同的 tile 作为源，避免单一 tile 主导，增加多样性。

#### 7.5.2 `tile()` — Padding 交换

```python
@staticmethod
def tile(latents_arr, step, groups, max_width=10, blend_mode="overwrite"):
    for key, val in groups.items():
        latent_source_id, side_source_id = map(int, key.split('_'))
        # 解析源 tile 和源方向

        # 轮询选择目标 tile
        candidate_idx = step % len_target
        chosen_latent_idx = target_latent_idx[candidate_idx]
        chosen_side_idx = target_side_idx[candidate_idx]

        source_latent = latents_arr[latent_source_id]
        target_latent = latents_arr[chosen_latent_idx]

        B, F, H, W = source_latent.pre_latent.size()
        # 从目标 tile 的 pre_latent 中取 padding 内侧的条带
        if chosen_side_idx == 0:  # Right
            tensor = target_latent.pre_latent[:, :, :, W - 2 * max_width: W - max_width]
        # ...

        rotation_value = TILING_ROTATION_MATRIX[side_source_id][chosen_side_idx]
        rotated_tensor = torch.rot90(tensor, rotation_value, [2, 3])
        # 旋转校正

        LatentHandler._write_padding_region(
            source_latent.post_latent, side_source_id,
            rotated_tensor, max_width, blend_mode=blend_mode,
        )
        # 将旋转后的切片写入源 tile 的 padding 区域
```

**`_write_padding_region` 支持两种模式**：

```python
if blend_mode == "overwrite":
    updated_region = new_tensor    # 直接覆盖
elif blend_mode == "weighted":
    weights = LatentHandler._build_blend_weights(current_region, side_idx)
    updated_region = (1.0 - weights) * current_region + weights * new_tensor
    # 线性加权融合：越靠近边缘权重越大，越靠近内容区域权重越小
```

**`_build_blend_weights` 详析**：
```python
if side_idx == 0:  # Right padding
    weights = torch.linspace(0.0, 1.0, width, ...)  # 从内到外递增
# 效果：padding 最外侧完全用新数据，最内侧保留原数据
# 好处：避免硬边界，使 padding 与内容区域的过渡更平滑
```

#### 7.5.3 `apply_random_padding_constraint()` — 随机 Padding 约束

```python
@staticmethod
def apply_random_padding_constraint(latents_arr, groups, max_width=10):
    # 只在最终推理完成后调用一次
    for key, val in groups.items():
        if len_target <= 1:
            continue
        # 随机选择一个 tile 作为源
        random_idx = random.randint(0, len_target - 1)
        source_latent_idx = target_latent_idx[random_idx]
        source_side_idx = target_side_idx[random_idx]

        latent = latents_arr[source_latent_idx]
        tensor_slice = get_tensor_slice_from_latent_and_side_idx(
            latent=latent.post_latent, side_idx=source_side_idx,
            max_width=0, max_replica_width=max_width
        )
        # 注意：这里 max_width=0, max_replica_width=max_width
        # 即取 padding 最外层 max_width 宽的条带

        for lat_idx, side_idx in zip(target_latent_idx, target_side_idx):
            if source_latent_idx == lat_idx:
                continue
            # 将源 tile 的 padding 复制到其他 tile 的 padding
            rotation_value = SIMILARITY_ROTATION_MATRIX[source_side_idx][side_idx]
            rotated_tensor_slice = torch.rot90(tensor_slice, rotation_value, [2, 3])
            current_latent.post_latent[:, :, :, W - max_width:] = rotated_tensor_slice
```

**为什么需要这一步？**  
在去噪循环中，不同 tile 的 padding 区域可能由于轮询选择机制导致最后一步的 padding 来源不一致。最终步额外做一次全局统一，确保解码前所有共享同边的 tile padding 完全一致。

### 8.2 MAG 计算原理

```python
def mean_absolute_gradient(img_1, img_2, direction, width_size=15):
    if direction == 'x':
        col_1 = img_1[:, -1]           # img1 最右列
        col_2 = img_2[:, 0]            # img2 最左列
        col_1_left = img_1[:, -width_size - 1]    # 更内侧参考
        col_2_left = img_1[:, -width_size]
        col_1_right = img_2[:, width_size + 1]
        col_2_right = img_2[:, width_size]

    gradients_1 = np.abs(col_1 - col_2)           # 拼接面梯度
    gradients_2 = np.abs(col_1_left - col_2_left)  # 内部参考梯度
    gradients_3 = np.abs(col_1_right - col_2_right)

    mean_gradient = max(mean_gradient_1, mean_gradient_2, mean_gradient_3)
    # 取三处梯度的最大值，更严格的评估
```

**原理**：完美拼接时，拼接面的梯度应该与图像内部的梯度一样平滑。MAG 值越低，表示拼接越自然。

### 8.3 Boundary SSIM 计算

使用局部高斯窗口计算 SSIM：
- 从拼接面取 `strip_width=15` 像素宽的条带
- 对每个通道分别计算 SSIM
- 取通道平均

$$\text{SSIM}(x, y) = \frac{(2\mu_x\mu_y + C_1)(2\sigma_{xy} + C_2)}{(\mu_x^2 + \mu_y^2 + C_1)(\sigma_x^2 + \sigma_y^2 + C_2)}$$

## 11. 复试高频问答预测

### Q1: 什么是 Diffusion Model（扩散模型）？它的前向和反向过程分别是什么？

**回答要点**：
- **前向过程（扩散）**：逐步向数据添加高斯噪声，经过 T 步后数据变为纯噪声：$q(x_t | x_{t-1}) = \mathcal{N}(x_t; \sqrt{1-\beta_t}x_{t-1}, \beta_t I)$
- **反向过程（去噪）**：学习一个神经网络 $\epsilon_\theta(x_t, t)$ 预测噪声，逐步从噪声恢复数据
- 训练损失：$L = \mathbb{E}_{t, x_0, \epsilon} [\| \epsilon - \epsilon_\theta(x_t, t) \|^2]$

### Q2: Latent Diffusion 与 Pixel-space Diffusion 的区别？

**回答要点**：
- Latent Diffusion 先用 VAE 将图像压缩到低维潜在空间（$512 \times 512 \rightarrow 64 \times 64 \times 4$），在潜在空间做扩散
- 优势：计算量大幅降低（$64^2 \times 4 \ll 512^2 \times 3$），且潜在空间已去除了不重要的高频细节
- 本项目的所有约束操作都在潜在空间进行

### Q3: 什么是 Classifier-Free Guidance (CFG)？

**回答要点**：
- CFG 不需要额外的分类器，而是通过同时训练条件模型和无条件模型实现引导
- 推理时：$\hat{\epsilon} = \epsilon_\theta(x_t, \emptyset) + w \cdot (\epsilon_\theta(x_t, c) - \epsilon_\theta(x_t, \emptyset))$
- $w$ 是 guidance scale，本项目设为 7.5
- 需要将 latent 复制 2 份（uncond + cond）做 batch forward

### Q4: 这个项目解决的核心技术挑战是什么？

**回答要点**：
- 在扩散去噪过程中保证**多个独立生成的 tile 之间边界无缝**
- 挑战：每个 tile 独立去噪，U-Net 只能看到单 tile 的内容，无法获取全局一致性
- 解决方案：通过 padding 区域交换实现跨 tile 的隐式通信，无需修改 U-Net 架构

### Q5: `max_width` 和 `max_replica_width` 的关系和 trade-off？

**回答要点**：
- `max_width`：padding 大小，决定 U-Net 能"看到"的邻居上下文范围
  - 太小 → 上下文不足，拼接边界有突变
  - 太大 → 显存爆炸，且 padding 区可能引入 artifact
- `max_replica_width`：相似性约束的复制宽度，padding 内侧子区域
  - 太小 → many-to-many 场景中共享同边的 tile 内容不一致
  - 太大 → 约束太强，降低图像多样性

### Q6: 为什么使用 `pre_latent` 和 `post_latent` 两个变量？

**回答要点**：
- `pre_latent`：存储 U-Net 去噪后和 similarity constraint 后的"干净"结果
- `post_latent`：在 `pre_latent` 基础上应用 tiling（padding 交换）后的版本
- 分开存储的原因：tiling 操作会修改 padding 区域，如果直接在 `pre_latent` 上修改，会影响下一步 similarity constraint 的源数据

### Q7: DDPM 和 DDIM 的区别？

**回答要点**：
- **DDPM**：马尔可夫链，每步去噪带随机性，$x_{t-1} = \mu_\theta(x_t, t) + \sigma_t z$，$z \sim \mathcal{N}(0,I)$
- **DDIM**：非马尔可夫，去噪过程是确定性的（$\sigma_t = 0$），支持非均匀步长跳步
- DDIM 优势：更少步数可达同等质量，且确定性生成利于复现
- 本项目默认用 DDPM（40 步），但支持 DDIM 和 Euler

### Q8: VAE 中 scaling_factor 的作用？

**回答要点**：
- `scaling_factor = 0.18215`
- VAE 编码后的 latent 分布方差较大，乘以该因子将其压缩到接近标准正态分布
- U-Net 是在这种缩放后的分布上训练的，所以编码后必须乘、解码前必须除
- 不做缩放会导致 U-Net 输入超出训练范围，生成质量严重下降

### Q9: 如何评估拼接质量？MAG 指标的原理？

**回答要点**：
- **MAG (Mean Absolute Gradient)**：计算拼接面两侧像素值的绝对差均值
- 完美拼接时 MAG 应接近图像内部的自然梯度水平
- 项目同时取拼接面和内部参考位置的梯度，取最大值作为更严格的评估
- 配合 Boundary SSIM（结构相似度）和 CLIP Score（语义一致性）综合评估

### Q10: 项目中 blend_mode 的设计考虑？

**回答要点**：
- `overwrite`：直接用邻居内容覆盖 padding，简单高效
- `weighted`：线性加权融合，靠近边界权重大、靠近内容区域权重小
  - 权重公式：`weights = torch.linspace(0.0, 1.0, width)`
  - 效果：平滑过渡，减少 padding 边界处的突变
  - 可能带来轻微的模糊效果

### Q11: 为什么旋转矩阵是必要的？

**回答要点**：
- 当一个 tile 的右边与另一个 tile 的上边拼接时，tensor 形状需要转置/旋转
- 例如右边 padding 形状为 `(B, F, H, max_width)`，上边 padding 形状为 `(B, F, max_width, W)`
- 旋转矩阵预计算了所有 4×4=16 种方向组合的旋转量，避免运行时条件判断

### Q12: 这个方法有什么局限性？

**回答要点**：
- padding 大小固定，无法自适应不同分辨率/内容
- 显存消耗随 `max_width` 增大而增加
- self-tiling 场景效果最好，many-to-many 场景中一致性约束可能限制多样性
- 仅在潜在空间施加约束，像素空间可能仍有微妙的接缝（需要大 padding 缓解）
- 不支持任意角度的拼接，仅限四方向（上下左右）

### Q14: 与 Seamless Tiling（circular padding）方法的区别？

**回答要点**：
- Seamless Tiling：修改 Conv2d 的 padding 为 circular，让网络"认为"图像在空间上是循环的
  - 优点：实现简单，效果好
  - 缺点：只支持 self-tiling，无法做 one-to-one 或 many-to-many
- Tiled Diffusion：通过显式的 padding 交换和约束
  - 优点：支持所有拼接场景，不修改网络结构
  - 缺点：额外计算开销，需要调参

### Q15: 代码中 `gc.collect()` 和 `torch.cuda.empty_cache()` 的作用？

**回答要点**：
- `gc.collect()`：Python 垃圾回收器强制回收不再引用的对象
- `torch.cuda.empty_cache()`：释放 PyTorch CUDA 缓存分配器持有的未使用的显存
- 两者配合使用：先 gc 回收 Python 层的 tensor 对象，再 empty_cache 释放底层 CUDA 内存
- 在每次 U-Net forward 之前调用，因为含 padding 的 latent 可能很大

### Q16: `torch.no_grad()` 的作用？

**回答要点**：
- 禁用自动梯度计算（autograd），推理时不需要反向传播
- 减少显存开销：不存储中间计算图
- 加速计算：跳过梯度追踪的额外操作
- 项目中所有推理操作（text encoder、U-Net、VAE 编码/解码）都在 `no_grad()` 上下文中
