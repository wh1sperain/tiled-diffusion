## 三、评价指标

| 指标 | 说明 | 方向 | 适用范围 |
| ---- | ---- | ---- | -------- |
| **CLIP Score** | 图文对齐度，使用 OpenAI CLIP (ViT-B/32) 计算图像与 prompt 之间的余弦相似度 | ↑ 越高越好 | 全部 |
| **MAG_x** | x 方向 Mean Absolute Gradient，衡量水平拼接边界处的梯度突变程度 | ↓ 越低越好 | 全部 |
| **MAG_y** | y 方向 Mean Absolute Gradient，衡量垂直拼接边界处的梯度突变程度 | ↓ 越低越好 | 全部 |
| **Boundary SSIM** | 相邻 tile 边界条带（宽 10px）的结构相似度，衡量拼接处的视觉连续性 | ↑ 越高越好 | many2many |
| **CLIP Consistency** | 各 tile 的 CLIP 嵌入向量之间的平均余弦相似度，衡量语义一致性 | ↑ 越高越好 | many2many |
| **Time (s)** | 单次生成耗时 | ↓ 越低越好 | 全部 |
| **VRAM (MB)** | 生成过程中的最大显存占用 | ↓ 越低越好 | 全部 |

> **注**：landscape 样例仅设置左右 self-tiling（`side_id=[1,1,None,None]`），因此其 MAG_y 为 N/A。Boundary SSIM 和 CLIP Consistency 仅在 many-to-many 模式下计算。

## 四、实验 A：max_width 消融

### 4.1 实验设计

- **消融变量**：`max_width` ∈ {4, 8, 12, 16, 32}
- **固定参数**：`max_replica_width` = 3
- **测试样例**：样例 1-5（self-tiling）
- **总运行数**：5 组 × 5 样例 = 25 次

`max_width` 控制 `LatentHandler.tile()` 中被复制到 padding 区域的最大像素宽度。数值越大，padding 区域越宽，生成的 latent 在边界处获得的上下文信息越多，但同时显存和计算开销也随之增加。

### 4.4 分析

1. **MAG 指标（拼接质量）**：
   - MAG_x 随 `max_width` 增大呈整体下降趋势，从 A-1 (mw=4) 的 0.0537 降至 A-4 (mw=16) 的 0.0365，表明更宽的 padding 区域有效减少了水平边界处的梯度突变。
   - MAG_y 同样在 mw=16 时达到最低值 0.0457，相比 mw=4 时的 0.0993 降低了 54%。
   - mw=32 的 MAG 值略高于 mw=16，说明过度扩展 padding 并不能带来进一步改善，反而可能因 padding 区域占比过大导致生成内容受限。

2. **CLIP Score（图文对齐度）**：
   - 各组 CLIP 均值在 0.3169–0.3336 之间，差异不大，说明 `max_width` 对语义质量影响有限。
   - 最高 CLIP 出现在 A-3 (mw=12, 0.3336)，但各组差异在统计波动范围内。

3. **资源消耗**：
   - 时间和显存随 `max_width` 单调递增。mw=32 的平均耗时（72.5s）为 mw=4（17.8s）的 4 倍，显存增加约 40%（5636→7906 MB）。
   - mw=16 到 mw=32 带来的 MAG 改善不明显，但资源消耗几乎翻倍，性价比显著下降。

4. **结论**：**`max_width = 16` 为最优值**，在拼接质量（最低 MAG_x=0.0365、MAG_y=0.0457）和资源消耗（35.8s、6465MB）之间取得最佳平衡。后续实验 B 固定 `max_width=16`。

## 五、实验 B：max_replica_width 消融

### 5.1 实验设计

- **消融变量**：`max_replica_width` ∈ {1, 3, 5, 8, 16}
- **固定参数**：`max_width` = 16（实验 A 最优值）
- **测试样例**：
  - Self-tiling：样例 1-5（5 个样例，与实验 A 相同）
  - Many-to-many：样例 6-8（3 个样例，2×2 拼接，每个生成 4 个 tile）
- **总运行数**：5 组 × (5 self-tiling + 3 many2many) = 40 次

`max_replica_width` 控制 Similarity Constraint 中 padding 区域保留的 replica 层数。该参数决定了相邻 tile 之间在扩散过程中共享信息的强度：值越小，约束越严格（直接赋值），tile 间一致性越高但可能导致生成自由度受限；值越大，约束越宽松，生成更自由但边界连续性可能降低。

### 5.6 分析

#### 5.6.1 Self-Tiling 模式

1. **MAG 指标（拼接质量）**：
   - **MAG_x** 在各组之间波动较小（0.0424–0.0556），没有呈现出明确的单调趋势。mrw=5 时 MAG_x 最低（0.0424），mrw=8 时最高（0.0556）。
   - **MAG_y** 同样在 0.0513–0.0683 之间波动，mrw=8 时最低（0.0513），mrw=16 时最高（0.0683）。
   - 整体来看，`max_replica_width` 对单 tile 的 self-tiling 拼接质量影响不显著，各组 MAG 差异在统计波动范围内。

2. **CLIP Score**：各组 CLIP 均值在 0.3023–0.3265 之间，差异不大，说明 `max_replica_width` 对语义生成质量无明显影响。

3. **资源消耗**：所有组的时间（31.9–33.7s）和显存（6465MB）基本一致，因为该参数仅影响 padding 区域的赋值策略，不改变计算图的规模。

#### 5.6.2 Many-to-Many 模式

1. **MAG 指标**：
   - **MAG_x**：mrw=1 时最低（0.0515），说明严格约束在控制梯度突变方面最有效。随 mrw 增大，MAG_x 略有上升但整体差异不大（0.0515–0.0616）。
   - **MAG_y**：mrw=1（0.0484）和 mrw=16（0.0487）最低，mrw=5 时最高（0.0714）。

2. **Boundary SSIM（边界结构相似度）**：
   - 呈现出随 mrw 增大而**上升**的趋势：mrw=1 为 0.8550，mrw=16 达到最高 0.8640。
   - 这表明更宽松的约束允许模型在边界区域学习到更平滑的过渡结构，提升了相邻 tile 边界处的视觉连续性。

3. **CLIP Consistency（语义一致性）**：
   - 同样随 mrw 增大而上升：mrw=3 最低（0.9503），mrw=16 最高（0.9714）。
   - 说明宽松约束下各 tile 的语义特征更加一致，模型有更多自由度来协调多个 tile 的整体风格。

4. **CLIP Score**：mrw=5 时最高（0.3458），显著高于 mrw=1（0.3287），表明适度宽松的约束有助于提升多 tile 场景下的图文对齐度。

5. **资源消耗**：各组时间（138.9–144.7s）和显存（6618MB）基本一致。

#### 5.6.3 Self-Tiling vs Many-to-Many 对比

| 对比维度 | Self-Tiling | Many-to-Many |
| -------- | ----------- | ------------ |
| mrw 对拼接质量 (MAG) 的影响 | 不显著，各组在统计波动范围内 | mrw=1 的 MAG 最低，但差异不大 |
| mrw 对边界质量的影响 | — | 较大 mrw 的 BSSIM 和 CLIPcons 更优 |
| 最优 mrw 选择 | 无明显倾向 | mrw=16 综合最优（BSSIM↑ CLIPcons↑） |
| 生成时间 | ~33s | ~141s（约 4.3 倍） |

#### 5.6.4 结论

`max_replica_width` 在 self-tiling 和 many-to-many 两种模式下呈现不同的影响特征：

- **Self-Tiling**：该参数对拼接质量影响有限，各取值下 MAG 差异在噪声范围内。这是因为 self-tiling 模式下所有方向的 padding 区域来自同一张图像的翻转/复制，约束强度的差异被自身纹理的一致性所弥补。

- **Many-to-Many**：该参数的影响更为显著。mrw=1（最严格约束）在 MAG 指标上表现最佳，但 mrw=16（最宽松约束）在 **Boundary SSIM**（0.8640）和 **CLIP Consistency**（0.9714）上取得最优值。这反映了一个 trade-off：严格约束直接控制梯度平滑度，而宽松约束给予模型更多自由度来生成结构上更连贯的边界过渡。

- **推荐配置**：考虑到 Boundary SSIM 和 CLIP Consistency 更能反映人类对拼接质量的感知（结构连续性优于单纯梯度平滑度），且 MAG 差异很小，推荐在 many-to-many 场景下使用 **mrw=16**，在 self-tiling 场景下 mrw 取值不敏感（默认值即可）。

## 六、综合结论

### 6.1 最优参数组合

| 参数 | 最优值 | 选择依据 |
| ---- | ------ | -------- |
| max_width | **16** | MAG_x 和 MAG_y 最低，资源消耗可控 |
| max_replica_width | **1** (self-tiling) / **16** (many-to-many) | self-tiling 下不敏感；many-to-many 下 BSSIM 和 CLIPcons 最优 |
| blend_mode | **overwrite** | MAG 指标全面优于 weighted；硬覆盖提供更强的约束信号 |

### 6.2 关键发现

1. **`max_width` 的 padding 宽度存在收益递减**：从 4→16 的 MAG 改善显著（MAG_x: 0.0537→0.0365, MAG_y: 0.0993→0.0457），但 16→32 的改善微乎其微甚至略有退化，而时间翻倍（35.8s→72.5s）、显存增加 22%（6465→7906 MB）。这表明存在一个"最优 padding 宽度"阈值，超过该阈值的额外上下文信息不再有助于减少边界突变。

2. **`max_replica_width` 在不同拼接模式下效果不同**：
   - **Self-tiling**：该参数在 1–16 范围内对 MAG 影响不显著（MAG_x: 0.0424–0.0556），说明对于单图自拼接场景，约束强度的变化不构成性能瓶颈。
   - **Many-to-many**：存在明确的 trade-off。mrw=1 的 MAG 最低（MAG_x=0.0515），但 mrw=16 的 Boundary SSIM（0.8640）和 CLIP Consistency（0.9714）最高。宽松约束给予模型更多自由度，使得多 tile 之间的边界过渡在结构层面更加自然。

3. **多角度评估的必要性**：仅使用 MAG 指标会偏向严格约束（mrw=1），但引入 Boundary SSIM 和 CLIP Consistency 后发现宽松约束在结构连续性和语义一致性方面更优。这说明边界质量是多维度的，需要综合多个指标来评估。

4. **CLIP Score 对 tiling 参数不敏感**：所有实验组的 CLIP 均值波动范围仅为 0.024（0.3023–0.3265），说明拼接参数主要影响边界质量而非语义内容。这符合直觉——padding 和 similarity 约束作用于 latent 空间的边缘区域，不影响中心区域的语义生成。

5. **不同纹理类型对参数敏感度不同**：
   - **geometric_pattern** (几何图案) 的 MAG 值普遍最低，因为规则图案天然具有良好的周期性。
   - **grass_field** (草地) 的 MAG_x 偏高，因为自然纹理的随机性使得精确匹配更困难。
   - **landscape** (风景) 的 MAG_x 一致偏低（0.010–0.019），该样例仅做左右 tiling，且天空/地面的水平连续性较易实现。
   - **wooden_floor_m2m** 在所有 mrw 下均保持极高的 BSSIM（>0.96）和低 MAG，表明方向性纹理（木板纹理）天然适合 tiling。

6. **Many-to-Many 2×2 拼接的资源开销可控**：时间约为 self-tiling 的 4.3 倍（~141s vs ~33s），符合预期（4 个 tile 各需一次完整扩散过程），但显存仅增加 2.4%（6618 vs 6465 MB），说明系统的显存管理效率较高。

7. **加权融合不优于硬覆盖（实验 C）**：尝试在 padding 区域使用线性权重融合（weighted）替代硬覆盖（overwrite），结果 MAG 指标全面退化（Self-Tiling MAG_x +14%，M2M MAG_x +56%），而 CLIP Score 和 CLIP Consistency 无改善。根因在于线性权重使 padding 内侧区域的约束趋近于零，有效约束宽度减半，U-Net 获得的邻居上下文信息被严重稀释。该实验验证了 **padding 区域的核心功能是提供完整的邻居上下文，硬覆盖是最直接有效的方式**。

## 七、实验 C：blend_mode 消融（加权融合 vs 硬覆盖）

### 7.1 实验设计

- **消融变量**：`blend_mode` ∈ {`overwrite`（硬覆盖）, `weighted`（加权融合）}
- **固定参数**：`max_width` = 16（实验 A 最优值），`max_replica_width` = 5
- **测试样例**：
  - Self-tiling：样例 1-5（与实验 A/B 共用）
  - Many-to-many：样例 6-8（2×2 拼接）
- **总运行数**：2 组 × (5 self-tiling + 3 many2many) = 16 次
- **实验编号**：C-1 = overwrite，C-2 = weighted

#### 方法说明

原始 Tiled Diffusion 的 `tile()` 函数在 padding 区域直接使用硬覆盖（overwrite）：将相邻 tile 的 latent 内容整体复制到当前 tile 的 padding 区域。这种方式在内容区域与 padding 区域的交界处存在突变——内容区域是 U-Net 的去噪结果，而 padding 区域是外部赋值的，二者在空间上不连续。

加权融合（weighted）策略尝试通过线性插值来缓解这一突变：在 padding 区域沿边界法线方向构建一个从 0 到 1 的线性权重梯度，使得：
- **内侧**（靠近内容区域）：$w \approx 0$，保留更多当前 latent 值（U-Net 预测结果的自然延续）
- **外侧**（靠近 tile 边缘）：$w \approx 1$，更多采用相邻 tile 的 latent 值

融合公式为：$\text{padding}_{\text{new}} = (1 - w) \cdot \text{padding}_{\text{current}} + w \cdot \text{padding}_{\text{neighbor}}$

### 7.6 分析

#### 7.6.1 代码正确性验证

经代码审查，加权融合的实现逻辑正确，不存在 bug：

1. **权重方向**：四个方向（Right/Left/Up/Down）的线性梯度均从内侧 ($w=0$) 到外侧 ($w=1$)，方向正确。
2. **融合公式**：$(1-w) \cdot \text{current} + w \cdot \text{neighbor}$ 保证了内侧保留更多当前值、外侧更多采用邻居值的语义。
3. **数据流**：`_write_padding_region` 正确读取 `post_latent` 的当前 padding 区域作为 current，使用 `pre_latent` 的邻居内容条带作为 neighbor，写回 `post_latent`。

**结论：效果不佳并非代码实现错误，而是该策略本身的局限性。**

#### 7.6.2 Self-Tiling 模式

1. **MAG 指标退化**：
   - **MAG_x**：weighted（0.0443）比 overwrite（0.0389）高 14%，边界梯度更明显。
   - **MAG_y**：weighted（0.0639）比 overwrite（0.0543）高 18%。
   - 大部分个体样例中 weighted 的 MAG 更高，说明加权融合整体上削弱了拼接质量。

2. **CLIP Score 基本无差异**：overwrite（0.3134）与 weighted（0.3135）几乎一致，证实 blend_mode 不影响语义生成质量。

3. **样例层面的混合表现**：grass_field 和 marble_stone 的部分 MAG 在 weighted 下更低，但 brick_wall 的 MAG_y 退化剧烈（0.0261 → 0.1345，+415%），说明效果高度依赖纹理类型。

#### 7.6.3 Many-to-Many 模式

1. **MAG_x 显著退化**：weighted（0.0862）比 overwrite（0.0552）高 **56%**，这是最主要的负面影响。尤其 wooden_floor 从 0.0182 升至 0.0850（+367%），说明原本极佳的水平拼接质量被大幅破坏。

2. **MAG_y 基本持平**：weighted（0.0826）与 overwrite（0.0822）相差无几。

3. **Boundary SSIM 有所提升**：weighted（0.2946）比 overwrite（0.2228）高 32%，但该提升主要由 wooden_floor（0.5552 vs 0.3521）贡献。在 brick_wall 上 SSIM 反而下降（0.1839 vs 0.2099）。

4. **CLIP Consistency 无显著差异**：0.9549 vs 0.9563，差距可忽略。

#### 7.6.4 根因分析：为什么加权融合效果不佳

加权融合的核心问题在于 **线性权重从 0 到 1 过渡削弱了 tiling 约束的有效范围**：

1. **有效约束宽度减半**：在 overwrite 模式下，整个 `max_width=16` 像素宽度的 padding 区域都被相邻 tile 的内容完全覆盖，提供了强约束信号。而在 weighted 模式下，内侧一半的权重 < 0.5，意味着这一区域主要保留的是当前 tile 自身的 latent 值，相邻 tile 的约束贡献很弱。**实质上，约束的有效作用宽度从 16 缩减到约 8 个 latent 像素**。

2. **约束信号的迭代衰减**：扩散过程有 40 步迭代，每步都重新应用 tiling 约束。在 overwrite 模式下，每步都将 padding 完全重置为邻居内容，约束是"清零重来"型的，不存在信号衰减。而在 weighted 模式下，内侧区域每步只混入很少的邻居信号 ($w \approx 0$)，40 步累积下来，该区域的 latent 值几乎完全由 U-Net 自由演化，缺乏来自邻居的引导。

3. **U-Net 感受野的上下文减少**：padding 的目的是为 U-Net 提供边界上下文。overwrite 模式下 U-Net 看到 16 像素宽的完整邻居内容，可以生成与之连贯的边界。weighted 模式下 U-Net 实际可用的邻居上下文被稀释，其在"看到"的信号中只有外侧较窄的区域是邻居信息占主导，内侧区域的邻居信息被当前 latent 的噪声所淹没。

4. **latent space 不同于 pixel space**：加权融合的直觉来自像素空间的 alpha blending，但 latent space 的特性不同。在 pixel space 中，线性混合两张图像确实能产生视觉上平滑的过渡。但在 latent space 中，简单的线性插值并不保证解码后的图像具有平滑过渡——VAE 的解码器是非线性映射，latent 值的线性混合可能映射到 pixel space 中的非线性、不自然的过渡。

5. **"平滑过渡"与"强约束"的矛盾**：加权融合试图在内容区—padding 区的交界处创造平滑过渡，但这和 tiling 的目标（使边界处的 latent 尽可能与邻居一致）存在根本矛盾。tiling 需要的是**强信号**而非**平滑信号**——U-Net 需要在 padding 区域看到邻居的真实内容才能生成与之匹配的边界。削弱这一信号虽然减少了 content-padding 交界处的突变，但代价是边界处的最终匹配质量下降。

#### 7.6.5 结论

| 对比维度 | overwrite (C-1) | weighted (C-2) | 差异 |
| -------- | --------------- | -------------- | ---- |
| Self-Tiling MAG_x | **0.0389** | 0.0443 | +14% ↑（退化） |
| Self-Tiling MAG_y | **0.0543** | 0.0639 | +18% ↑（退化） |
| M2M MAG_x | **0.0552** | 0.0862 | +56% ↑（显著退化） |
| M2M BSSIM | 0.2228 | **0.2946** | +32% ↑（改善） |
| CLIP Score | 0.3134 | 0.3135 | 无差异 |
| CLIP Consistency | 0.9563 | 0.9549 | 无差异 |

**加权融合策略整体上不优于原始硬覆盖方式。** 其在 MAG 指标上的显著退化表明，削弱 padding 区域的约束强度会导致拼接边界处的梯度突变增加。虽然 Boundary SSIM 在个别样例（wooden_floor）上有所提升，但这一改善不具有普遍性，且被 MAG 的全面退化所抵消。

本实验验证了一个重要认知：**在 Tiled Diffusion 框架中，padding 区域的核心作用是为 U-Net 提供尽可能完整、真实的邻居上下文信息。任何稀释或削弱该信息的操作都会损害最终的拼接质量。** 原始论文采用硬覆盖是合理且有效的设计选择。

## 八、复试常见问题与解答

### 7.1 数据处理相关

**Q1：图像生成中为什么要在 latent space 而非 pixel space 进行扩散？**

在 pixel space 中直接对高分辨率图像（如 512×512×3）做扩散过程，每一步都需要处理完整的像素级张量，计算量和内存开销极大。Latent Diffusion Model (LDM) 先通过一个预训练的 VAE (Variational Autoencoder) 将图像编码到低维 latent space（如 64×64×4），然后在该空间中执行扩散过程。这样做的好处：

- **计算效率**：latent 维度远小于像素维度（压缩比约 48:1），极大降低了训练和推理成本。
- **语义紧凑性**：latent space 编码了图像的高级语义信息，扩散过程在语义空间而非像素空间中工作，有利于生成结构合理的图像。
- **可扩展性**：同一 latent 扩散框架可通过更换 VAE 来适配不同分辨率和模态。

**Q2：Tiled Diffusion 中如何处理 tile 之间的边界？**

Tiled Diffusion 使用两种核心机制来保证 tile 之间的边界连续性：

1. **Padding（`max_width` 参数）**：在 `LatentHandler.tile()` 中，每个 tile 的 latent 不仅包含自身区域，还会在上下左右四个方向扩展 padding 区域。padding 区域填充了相邻 tile（或自身翻转）的 latent 值，使得 U-Net 在处理每个 tile 时能"看到"边界附近的上下文，从而生成更连续的内容。
2. **Similarity Constraint（`max_replica_width` 参数）**：在每一步去噪后，约束会检查并修正 padding 区域的 latent 值。`max_replica_width` 控制保留多少层 replica（模型预测的值），其余部分用相邻 tile 的实际值替换。这确保了前向传播时边界区域的 latent 一致。

**Q3：什么是 Mean Absolute Gradient (MAG)？为什么用它来评估拼接质量？**

MAG 是 Tiled Diffusion 拼接质量的核心量化指标。计算方法：将生成的图像在水平（或垂直）方向做 2×2 tiling 拼成大图，然后在拼接边界位置计算像素值的绝对梯度均值。公式为 $\text{MAG} = \frac{1}{H} \sum_{i=1}^{H} |I(i, x_{\text{seam}}) - I(i, x_{\text{seam}}-1)|$，其中 $x_{\text{seam}}$ 是拼接线位置。

MAG 直接度量了拼接线处的亮度/颜色突变程度。若 tiling 完美无缝，边界处的梯度应与图像内部的自然梯度一致，MAG 值较低。若存在明显接缝，梯度会出现突变，MAG 升高。

**Q4：Boundary SSIM 和 CLIP Consistency 分别衡量什么？**

- **Boundary SSIM**：取相邻 tile 拼接边界处各 10 像素宽的条带，计算两者的 SSIM (Structural Similarity Index)。SSIM 综合考虑亮度、对比度和结构三个方面的相似性，比单纯的像素差异更接近人类视觉感知。值越接近 1 说明边界处的视觉过渡越平滑。

- **CLIP Consistency**：对 many-to-many 模式下的各个 tile 分别用 CLIP 模型提取图像嵌入向量，然后计算所有 tile 对之间的余弦相似度均值。该指标衡量的是多个 tile 在语义/风格层面的一致性——即使内容不完全相同，它们是否在整体风格和语义上保持统一。

### 7.2 扩散模型基本概念

**Q5：解释 DDPM 的前向和反向过程。**

DDPM (Denoising Diffusion Probabilistic Models) 包含两个过程：

- **前向过程（加噪）**：给定原始数据 $x_0$，通过渐进添加高斯噪声得到一系列噪声越来越大的中间状态 $x_1, x_2, ..., x_T$。每一步定义为 $q(x_t | x_{t-1}) = \mathcal{N}(x_t; \sqrt{1-\beta_t} x_{t-1}, \beta_t I)$，其中 $\beta_t$ 是预定义的噪声调度。当 $T$ 足够大时，$x_T$ 近似标准高斯噪声。

- **反向过程（去噪）**：训练一个神经网络 $\epsilon_\theta(x_t, t)$ 来预测每一步中添加的噪声。推理时从纯噪声 $x_T \sim \mathcal{N}(0, I)$ 出发，逐步去噪得到干净的图像 $x_0$。每一步的反向转移为 $p_\theta(x_{t-1}|x_t) = \mathcal{N}(x_{t-1}; \mu_\theta(x_t, t), \sigma_t^2 I)$。

**Q6：DDPM 和 DDIM 的区别是什么？**

- **DDPM** 的反向过程是随机的（每步添加随机噪声），且通常需要 1000 步才能生成高质量图像。
- **DDIM** (Denoising Diffusion Implicit Models) 将反向过程改写为确定性形式，不再每步添加随机噪声。核心优势是支持跳步采样（如从 1000 步压缩到 50 步），大幅加速推理速度，同时保持生成质量。DDIM 还支持在 latent space 中做插值（因为过程是确定性的）。

**Q7：Classifier-Free Guidance (CFG) 的原理是什么？`cfg_scale` 怎么理解？**

CFG 是一种无需额外分类器的条件引导方法。训练时随机丢弃条件信号（如 text prompt）来同时学习条件模型 $\epsilon_\theta(x_t, t, c)$ 和无条件模型 $\epsilon_\theta(x_t, t, \varnothing)$。推理时对两者做线性外推：

$$\hat{\epsilon} = \epsilon_\theta(x_t, t, \varnothing) + w \cdot [\epsilon_\theta(x_t, t, c) - \epsilon_\theta(x_t, t, \varnothing)]$$

其中 $w$ 即 `cfg_scale`。$w=1$ 等价于标准条件生成，$w>1$ 会增强条件信号的引导强度，使生成结果更忠实于 prompt 但多样性下降。$w=7.5$ 是 SD 1.5 的常用值，在质量和多样性之间取得平衡。

**Q8：U-Net 在扩散模型中的作用是什么？**

U-Net 是扩散模型的主干网络，负责预测每一步去噪所需的噪声 $\epsilon_\theta(x_t, t)$。其结构特点：

- **编码器-解码器架构**：编码器逐层下采样提取多尺度特征，解码器逐层上采样恢复分辨率。
- **跳跃连接 (Skip Connections)**：将编码器的中间特征直接连接到解码器的对应层，保留细节信息。
- **时间步嵌入 (Timestep Embedding)**：将当前时间步 $t$ 编码为向量并注入网络各层，使模型感知当前的噪声水平。
- **交叉注意力 (Cross-Attention)**：在 SD 等文本引导模型中，U-Net 通过交叉注意力机制融合文本嵌入（来自 CLIP text encoder），实现文本到图像的语义引导。

### 7.3 项目技术细节

**Q9：Tiled Diffusion 的核心思想和技术路线是什么？**

Tiled Diffusion 的目标是生成可无缝平铺 (seamless tiling) 的纹理图像。核心思想：**在扩散过程中施加空间约束，使图像的对边（左右、上下）在 latent space 中保持一致性**。

技术路线分三步：
1. **Tiling Constraint**：将 latent 在各方向上做 padding（复制/翻转自身或相邻 tile 的 latent），扩展 U-Net 的感受野，使其在去噪时能看到边界处的完整上下文。
2. **Similarity Constraint**：每步去噪后，对 padding 区域的预测值和相邻 tile 的实际值做混合/替换，确保边界处的 latent 表示趋向一致。
3. **Random Padding Constraint**：对非 tiling 方向的 padding 区域填充随机噪声（且每步噪声强度递减），避免非 tiling 方向出现不自然的对称伪影。

**Q10：Self-tiling 和 Many-to-many 拼接有什么区别？**

- **Self-tiling**：一张图像与自身拼接（类似墙纸效果），要求图像的四边（或两边）能完美对齐。实现上，padding 区域填充的是自身 latent 的翻转/复制。
- **Many-to-many**：多张不同图像组成网格（如 2×2），相邻 tile 之间的边界需要平滑过渡。实现上，padding 区域填充的是真正相邻 tile 的 latent，Similarity Constraint 在相邻 tile 之间传递信息。

Many-to-many 的难度更高，因为各 tile 是独立生成的不同图像，需要在保持各自语义的同时实现边界连贯。

**Q11：`max_width` 和 `max_replica_width` 参数在代码中是如何实现的？**

- **`max_width`**：在 `LatentHandler.tile()` 中使用。该函数将 latent 张量在四个方向（上下左右）做 padding 扩展。padding 宽度为 `min(max_width, latent_size)`，即不超过 latent 本身的尺寸。padding 内容取自相邻 tile 的 latent（many-to-many 模式）或自身翻转（self-tiling 模式）。

- **`max_replica_width`**：在 `apply_similarity_constraint()` 中使用。每步去噪后，padding 区域的 latent 会被重新赋值：距离 tile 边界 ≤ `max_replica_width` 的部分保留模型预测值（replica），其余部分用相邻 tile 的实际 latent 值替换。直觉上，`max_replica_width` 越小，约束越强——极端情况 `mrw=1` 意味着 padding 区域几乎全部被相邻 tile 的值覆盖。

**Q12：实验中观察到的主要现象和结论是什么？**

1. **padding 宽度（max_width）存在最优点**：mw=16 在拼接质量和资源消耗之间取得最佳平衡。进一步增大到 32 的 MAG 改善微乎其微，但时间和显存大幅增加。
2. **约束强度（max_replica_width）的影响取决于拼接模式**：self-tiling 下不敏感；many-to-many 下严格约束降低 MAG，但宽松约束提高 Boundary SSIM 和 CLIP Consistency。
3. **评价指标的多维性**：MAG 衡量梯度平滑度，BSSIM 衡量结构连续性，CLIPcons 衡量语义一致性——三者可能指向不同的最优参数，需要综合权衡。
4. **纹理类型影响拼接难度**：规则图案（geometric_pattern）天然适合 tiling，自然纹理（grass_field）挑战最大。

### 7.4 深度学习与评价指标基础

**Q13：CLIP 模型的工作原理是什么？**

CLIP (Contrastive Language-Image Pre-training) 是 OpenAI 提出的多模态模型，通过对比学习在大规模图文数据上训练。核心结构包含两个编码器：

- **Image Encoder**（如 ViT-B/32）：将图像编码为固定长度的嵌入向量。
- **Text Encoder**（Transformer）：将文本编码为同一嵌入空间的向量。

训练目标：使匹配的图文对嵌入向量的余弦相似度最大化，不匹配的最小化（InfoNCE loss）。训练后，CLIP 具备零样本（zero-shot）的图文匹配能力。在本项目中，CLIP Score = $\cos(\text{img\_emb}, \text{text\_emb})$，用于评估生成图像与 prompt 的语义对齐程度。

**Q14：SSIM 和 MSE 相比有什么优势？**

MSE (Mean Squared Error) 仅考虑逐像素的数值差异，对亮度偏移、对比度变化和结构失真一视同仁。SSIM (Structural Similarity Index) 将图像质量分解为三个独立分量：

$$\text{SSIM}(x, y) = l(x,y)^{\alpha} \cdot c(x,y)^{\beta} \cdot s(x,y)^{\gamma}$$

其中 $l$ 衡量亮度（均值比较），$c$ 衡量对比度（方差比较），$s$ 衡量结构（协方差比较）。

SSIM 的核心优势在于其对结构信息的敏感度更接近人类视觉系统——人眼对结构失真（如模糊、鬼影）比对均匀亮度偏移更敏感，而 MSE 无法区分这两者。

**Q15：VAE（变分自编码器）的原理和在 Stable Diffusion 中的作用？**

VAE 由编码器和解码器组成。编码器将输入映射到一个概率分布（通常是高斯分布的均值 $\mu$ 和方差 $\sigma^2$），然后从该分布中采样得到 latent 向量 $z$（通过重参数化技巧 $z = \mu + \sigma \cdot \epsilon$，其中 $\epsilon \sim \mathcal{N}(0, I)$）。解码器将 $z$ 重构回输入空间。训练损失 = 重构损失 + KL 散度（使后验分布接近标准正态先验）。

在 Stable Diffusion 中，VAE 的作用：
- **编码**：将 512×512×3 的像素图像压缩到 64×64×4 的 latent 表示，扩散模型在此空间工作。
- **解码**：扩散完成后将生成的 latent 解码回像素空间，得到最终图像。
- VAE 本身是预训练好的，在推理阶段权重冻结。
